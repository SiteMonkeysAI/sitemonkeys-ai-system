Issue #881: [claude-fix] External lookup fails to trigger on conversational and follow-up query phrasing

Issue Description:
Problem
External lookup is not triggering on queries that are phrased conversationally or as follow-up questions, even when the subject matter is highly volatile and the information is clearly time-sensitive. Confirmed failures from live testing:

"Seems like Elon Musk has something going on, what is it" — no external lookup fired. System gave a generic response about SpaceX/Tesla ventures from training data. Reality: major breaking news this week including Cybercab production start, FSD March 31 deadline, AGI declaration, xAI environmental scandal.
"Did Saudi Arabia make a big commitment" — no external lookup fired. System said it couldn't confirm outcomes. Reality: this refers to one of the most documented events of 2025 — Saudi Arabia's $600 billion investment commitment to the US during Trump's May 2025 Gulf tour, later expanded to nearly $1 trillion in November 2025, including the largest defense sales agreement in history at $142 billion.
"What is Schumer demanding from Trump" — no external lookup fired despite being a clearly current political event.
"Did the Coast Guard have anything really big happen" — no external lookup fired.

Root cause
The freshness marker detection in truthTypeDetector.js (and wherever trigger detection lives in externalLookupEngine.js) relies on specific keywords like "current", "latest", "today", "price", "what's happening with." Conversational phrasing bypasses all of these:

"seems like... has something going on" — no marker
"did [X] make a big commitment" — no marker
"did [X] have anything really big happen" — no marker
"what is [person] demanding" — no marker
Follow-up queries mid-conversation inherit no freshness context from the prior turn

The fix must use semantic understanding, not expanded keyword lists. Adding more keywords is whack-a-mole and will never cover natural human phrasing. The classifier needs to understand that any query about a named public figure's recent actions, any query about a named political or geopolitical event, and any follow-up question building on a previous volatile topic should trigger external lookup by default.
Specific behaviors required
Any query matching these semantic patterns must trigger external lookup:

Named public figure + any action verb in present or recent past tense ("has something going on", "doing anything", "what happened with", "any news on")
Named country/government + commitment/action/deal/announcement
Named politician + current demands/positions/actions
Named organization + recent incident/event
Follow-up queries (short questions building on prior context) where the prior turn triggered external lookup — the follow-up should inherit the volatile classification

Do not expand the keyword list as the primary fix. The solution must be semantic classification that understands intent, not pattern matching on surface phrases. If the query is about what a real named entity is doing in the present or recent past, external lookup fires.
