## Changes Made

### Issue F: Authentication guards for sensitive endpoints

**`api/admin/cleanup.js`** — `POST /api/admin/cleanup-stale-memories`
- Added `ADMIN_KEY` auth check at the top of `handleCleanupRequest`, matching the pattern used in `cleanup-zombies.js`
- Checks `x-admin-key` header and `Authorization: Bearer <token>` header
- Returns `403 Unauthorized` if missing or incorrect

**`api/system-status.js`** — `GET /api/system-status`
- Added `INTERNAL_ACCESS_TOKEN` auth check at the top of the `systemStatus` handler
- Checks `x-internal-token` header and `Authorization: Bearer <token>` header
- Returns `403 Unauthorized` if missing or incorrect

**`server.js`** — `GET /api/run-tests` and `GET /api/memory/list`
- Added identical inline `INTERNAL_ACCESS_TOKEN` auth guards at the top of both inline handlers
- Both check `x-internal-token` header and `Authorization: Bearer <token>` header
- Return `403 Unauthorized` if missing or incorrect
- No frontend references to these endpoints were found (confirmed via codebase search), so adding auth will not break existing callers

### Issue G: File upload type restriction — `api/upload-file.js`
- Added `ALLOWED_MIME_TYPES` allowlist: PDF, Word (.doc/.docx), plain text, CSV, JSON
- Added `ALLOWED_EXTENSIONS` allowlist: `.pdf`, `.doc`, `.docx`, `.txt`, `.csv`, `.json`
- Replaced permissive `cb(null, true)` fileFilter with explicit extension + MIME type validation
- Rejected files return an error: `"File type not allowed. Accepted types: PDF, Word documents, text files, CSV, JSON"`
- Reduced `fileSize` limit from 50MB to 10MB
