Issue #939: [Performance] Add HNSW index on embedding column — memory retrieval currently does full sequential scans loading ~6MB per query

Issue Description:
## Problem
 
The audit confirmed there is no vector index on the embedding column
in the memories table. Every semantic memory retrieval query currently:
 
1. Loads up to 500 rows with full embedding vectors into Node.js
2. Computes cosine similarity in JavaScript
3. Transfers approximately 6MB of embedding data per query over
   the wire
 
This works at current scale but degrades directly with memory
growth. As the 3-6M token memory system grows, every query gets
slower and more expensive. Adding a proper index moves similarity
search into PostgreSQL where it belongs and keeps retrieval fast
regardless of how large memory grows.

## Required Fix
 
### Step 1 — Create the migration file
 
Create a new file: `sql/add_embedding_index.sql`
 
Contents:
 
  -- Add HNSW index for fast approximate nearest neighbor search
  -- on the embedding column in the memories table
  --
  -- HNSW is preferred over IVFFLAT for this use case because:
  -- - No training/clustering step required
  -- - Better query performance at this scale
  -- - Supports incremental inserts without rebuild
  --
  -- m=16, ef_construction=64 are standard starting values
  -- for tables under 1M rows. Adjust upward if needed later.
 
  CREATE INDEX CONCURRENTLY IF NOT EXISTS
    memories_embedding_hnsw_idx
  ON memories
  USING hnsw (embedding vector_cosine_ops)
  WITH (m = 16, ef_construction = 64);
 
  -- Verify index was created
  SELECT indexname, indexdef
  FROM pg_indexes
  WHERE tablename = 'memories'
  AND indexname = 'memories_embedding_hnsw_idx';

  export async function addEmbeddingIndex(req, res) {
    const adminKey = req.headers['x-admin-key'] ||
      req.headers['authorization']?.replace('Bearer ', '');
 
    if (!adminKey || adminKey !== process.env.ADMIN_KEY) {
      return res.status(403).json({ error: 'Unauthorized' });
    }
 
    try {
      console.log('[EMBEDDING-INDEX] Starting HNSW index creation...');
      console.log('[EMBEDDING-INDEX] This may take several minutes
        on large tables. CONCURRENTLY means no table lock.');
 
      const sql = readFileSync('./sql/add_embedding_index.sql', 'utf8');
      await pool.query(sql);
 
      console.log('[EMBEDDING-INDEX] ✅ Index created successfully');
      return res.json({
        success: true,
        message: 'HNSW index created on memories.embedding',
        index_name: 'memories_embedding_hnsw_idx'
      });
    } catch (error) {
      console.error('[EMBEDDING-INDEX] ❌ Error:', error.message);
      return res.status(500).json({
        success: false,
        error: error.message
      });
    }
  }
