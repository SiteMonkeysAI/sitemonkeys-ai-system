Issue #883: [claude-fix] Two remaining external lookup failures: institutional abbreviations not recognized as proper nouns, and conversational queries sent raw to news APIs producing weak results

Issue Description:
Problem
Two failures remain after PR #881:
Failure 1 — "The Fed" and similar institutional abbreviations not triggering lookup
Query tested: "Seems like the Fed has something going on lately" — no lookup fired, hollow response returned.
Root cause: The hasNamedEntityActionPattern() proper noun detector introduced in PR #881 identifies proper nouns by looking for capitalized words. "The Fed" fails this check because "the" is lowercase and "Fed" alone may not score high enough, or the phrase "has something going on lately" doesn't match the exact phrase "has something going on" in the pattern. Either way the result is the same — a clearly current-events query about a major institution gets classified as non-volatile and no lookup fires.
Common institutional abbreviations that must be recognized as named entities regardless of capitalization context: Fed, FBI, CIA, NSA, NATO, UN, EU, IMF, WHO, CDC, IRS, SEC, DOJ, DHS, Pentagon, Congress, Senate, Supreme Court, White House, and similar. These are always referring to real institutions whose current activities are volatile by definition.
Failure 2 — Conversational queries passed raw to news APIs, returning weak or irrelevant results
Query tested: "Did Saudi Arabia make a big commitment to the US?" — lookup fired correctly, but returned thin recent results (FedEx expansion, naval exercises) instead of the landmark $600 billion investment commitment and $142 billion arms deal from Trump's May 2025 Gulf tour.
Root cause: The external lookup engine is passing the user's conversational question directly as the search query to NewsAPI and Serper. Conversational phrasing like "did Saudi Arabia make a big commitment" is a poor search query — news APIs are optimized for keyword and topic searches, not natural language questions. The result is that the most relevant and important articles are missed while tangentially related recent articles surface instead.
The engine needs to reframe the conversational query into an optimized news search query before hitting the APIs. For example:

"Did Saudi Arabia make a big commitment to the US" → search query: "Saudi Arabia US investment commitment"
"Seems like Elon Musk has a lot going on" → search query: "Elon Musk latest news"
"What is Schumer demanding from Republicans" → search query: "Schumer Republican demands"
"Did China respond" (follow-up) → search query: "China response Trump tariffs" (using context from prior turn)

Fix 1 — Institutional abbreviation recognition
In hasNamedEntityActionPattern() in truthTypeDetector.js and hasProperNouns() / isCurrentEventQuery() in externalLookupEngine.js, add a curated list of known institutional abbreviations and short names that always count as named entities. The check should be: does the query contain any word from this list OR any capitalized proper noun? Either condition satisfies the named entity requirement.
This list should include at minimum: Fed, FBI, CIA, NSA, NATO, UN, EU, IMF, WHO, CDC, IRS, SEC, DOJ, DHS, Pentagon, Congress, Senate, SCOTUS, White House, Kremlin, Vatican, OPEC, FDIC, CFPB, and reasonable additions the agent identifies in the codebase.
Fix 2 — Query rewriting before news API calls
In externalLookupEngine.js, before passing a query to NewsAPI, Serper, or TheNewsAPI, run it through a query rewriting step when the query is classified as a conversational current-event query (isCurrentEventQuery returns true or entity_action_current_event pattern was matched).
The rewriting should:

Strip question structure ("did", "does", "what is", "seems like", "has something going on")
Extract the core subject (named entity) and action/topic
Construct a compact 3-5 word keyword search query optimized for news APIs
For follow-up queries, incorporate the subject from the prior turn's context if the follow-up lacks its own named entity

This does not need to be an AI call — a deterministic extraction that removes question words and stopwords and retains proper nouns plus key action words will produce dramatically better results than passing the raw conversational query. If the agent determines an AI-assisted rewrite produces meaningfully better results at acceptable token cost, that is acceptable, but a deterministic approach should be tried first.
