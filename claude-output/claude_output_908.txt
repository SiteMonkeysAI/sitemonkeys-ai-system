## Changes Made to `api/core/intelligence/externalLookupEngine.js`

### Fix 1: GoldAPI key injection (confirmed working, improved extract)
The `x-access-token` header was already being sent via `getHeaders()` in `performLookup`. Confirmed correct. Updated the GoldAPI `extract` function to return a verbose 200+ character response (including change, ask/bid, gram price, context text) so it passes the 200-char minimum content threshold that was silently blocking price responses.

### Fix 2: Added metals.live as free no-key fallback for gold and silver
Added `API_SOURCES.COMMODITIES[2]` — a new `metals.live` source using `fetchData` (bypasses the 200-char threshold check which doesn't apply to `fetchData` sources). Endpoints: `https://api.metals.live/v1/spot/gold` and `https://api.metals.live/v1/spot/silver`. No API key or authentication required. Handles both regular spot price queries and weight-based queries.

### Fix 3: Replaced CoinGecko with Coinbase as primary crypto source
Inserted Coinbase as `API_SOURCES.CRYPTO[0]` using `fetchData`. Fetches `https://api.coinbase.com/v2/prices/BTC-USD/spot` and/or `https://api.coinbase.com/v2/prices/ETH-USD/spot` based on query content (detects bitcoin/btc, ethereum/eth, or fetches both for generic crypto queries). Parses `{"data":{"amount":"...", ...}}` format. CoinGecko moved to position 1 as fallback with improved extract (200+ chars).

### Fix 4: Unit conversion for weight-based price queries
Built into the `metals.live` `fetchData` source. When a query contains value-intent keywords (worth/value/cost/price/how much) AND a weight pattern (e.g. "50 lbs", "2 kg", "10 ounces"), it:
1. Converts the quantity to troy ounces using the specified constants (1 lb = 14.5833 troy oz, 1 kg = 32.1507 troy oz, 1 gram = 0.0321507 troy oz, 1 oz avoirdupois = 0.911458 troy oz)
2. Fetches the live spot price from metals.live
3. Returns the full math: `"Gold weight calculation: 50 lbs = 729.17 troy oz × $3,247.50/troy oz = $2,368,000 USD total value..."`

### selectSourcesForQuery update
The commodity detection block now unconditionally adds `API_SOURCES.COMMODITIES[2]` (metals.live) after the key-gated Metals-API and GoldAPI sources. This ensures metals.live is always available as a cascade fallback when the keyed sources fail, return 429 errors, or when no keys are configured.

---END SUMMARY---
