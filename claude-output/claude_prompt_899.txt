## Problem

Two distinct bugs prevent Federal Reserve queries from returning current news. Both confirmed via Railway logs on 2026-03-11.

---

## Bug 1 — Wrong source routing (CoinGecko instead of NewsAPI)

**Query:** `"Seems like the fed has something going on lately"`

**Log evidence:**
willAttemptLookup: true
isSemanticCurrentEventQuery: true
[externalLookupEngine] Sources: CoinGecko
[externalLookupEngine] CoinGecko returned only 35 chars (below 200 threshold), cascading to next source
[externalLookupEngine] Lookup failed, executing graceful degradation

The trigger fires correctly and the institution is recognized, but the query is being routed to CoinGecko (crypto pricing source) instead of NewsAPI. CoinGecko returns 35 chars of crypto price data, fails the minimum threshold, and the entire lookup fails. The source selection logic is misclassifying this as a financial/crypto query rather than a news query.

**Root cause:** The source routing decision in `externalLookupEngine.js` is selecting CoinGecko based on something in the query (likely "fed" matching a financial keyword pattern) before the institution recognition has a chance to route it correctly to news sources.

---

## Bug 2 — Trigger detection miss

**Query:** `"What's the fed doing these days"`

**Log evidence:**
willAttemptLookup: false
isSemanticCurrentEventQuery: false

The named entity + action pattern added in PR #881 (`hasNamedEntityActionPattern()`) is not matching this phrasing. "What's the fed doing" is structurally different from the patterns that were added — the contraction "What's" and informal structure causes a miss.

**Root cause:** `hasNamedEntityActionPattern()` in `truthTypeDetector.js` needs to cover contraction-based phrasings and informal "what's X doing" structures when X is a known institution.
