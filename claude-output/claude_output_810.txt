## Changes Made (Issue #810 — Systemic Fixes)

### Files Modified: 4

---

### 1. `api/core/intelligence/externalLookupEngine.js`

**Fix E (P2): Weather misclassification (false positives)**
- Changed `/weather|temperature|forecast|rain|snow|storm/i` to `/\b(weather|temperature|forecast|rain|snow|storm)\b/i`
- **Root cause**: `/rain/i` matched "Ukraine" as a substring (u-k-**r-a-i-n**-e), triggering the weather detection code path for non-weather queries like "Is there anything new in the Ukraine war with Russia"

**Fix D (P1): Raw conversational query sent to RSS**
- Modified the `buildUrl` function for the stock price RSS fallback source: instead of appending the raw user query (e.g., "What is the current stock price of Walmart") to the RSS URL, it now strips question/price vocabulary words to extract just the entity name (e.g., "Walmart"), producing cleaner search queries like "Walmart stock price"
- Applied same entity extraction approach to the commodity price RSS fallback source

---

### 2. `api/core/personalities/eli_framework.js`

**Fix A (P0): Business boilerplate leaking on non-business queries**
- Added `isNonBusinessQueryType` detection using `truthType`, `analysis.intent`, and `context.queryClassification.classification`
- Gates: STEP 1 (risk analysis), STEP 2 (assumption challenges), STEP 3 (downside scenarios), STEP 4 (survival metrics), STEP 5 (blind spots) now all check `!isNonBusinessQueryType`
- Queries classified as `market_query`, `current_events`, `news`, or `DOCUMENT_REVIEW` will never receive business boilerplate sections ("⚠️ Critical Risks I See", "💰 Business Survival Analysis", "🤔 Assumptions to Challenge", "🔍 What We're Not Seeing Yet") even when mode is `site_monkeys` or `business_validation`

---

### 3. `api/core/orchestrator.js`

**Fix B (P1): RSS-only price responses still say "price not provided"**
- Added "Position 3: Market Query RSS Clamp" in the Layer 2 fallback primitives section (runs after AI generation, before enforcement chain)
- Detects when: (a) the external data source was RSS/news (`phase4Metadata.fetched_content` contains `[DATA TYPE: NEWS HEADLINES`), AND (b) the query was market-related
- If the AI response still contains forbidden phrases like "price not provided/given/available", replaces them with "no live quote API is configured for this asset" and appends the canonical disclosure pointing to Google Finance / Yahoo Finance
- This is a code-level deterministic fix, not prompt-dependent

---

### 4. `api/core/intelligence/queryComplexityClassifier.js`

**Fix G (P2): Embedding token limit exceeded on large documents**
- Added two-layer truncation guard in `getCachedEmbedding()`: hard limit of 25,000 characters (≈ 6,250 tokens with headroom below the 8,192-token limit)
- Added intent-focused truncation in `classifyQueryComplexity()`: uses only the first 500 characters of the query for classification (the user's question, not the entire pasted document)
- **Root cause**: When users paste large documents into the chat message field (e.g., a 5,594-word hardscaping document), the full `query` string was being sent to the embedding API, causing `BadRequestError: 400 This model's maximum context length is 8192 tokens, however you requested 8913 tokens`
