## Issue #759 - CMP2: International names lost during memory compression

### Problem Analysis
When a user sends a message containing international names with special characters (diacritics, hyphens, apostrophes), such as:
- "Dr. Xiaoying Zhang-Müller"
- "Björn O'Shaughnessy"
- "José García-López"

The system's compression step (extractKeyFacts in intelligent-storage.js) uses an LLM to compress messages. The LLM sometimes strips or simplifies these special characters, converting:
- "José" → "Jose"
- "Björn" → "Bjorn"
- "García-López" → "Garcia Lopez"

This causes the names to be lost during retrieval because the stored form doesn't match the original.

### Root Cause
The code had protection mechanisms for:
1. High-entropy tokens (ALPHA-123-ABC format identifiers)
2. Brand names (Tesla Model 3)
3. Numbers and dates

But it did NOT have protection for international names with special characters. The existing `extractUnicodeNames` method (which properly extracts names with special characters) was not being used to verify that names survived the LLM compression process.

### Solution Implemented

Modified `/api/memory/intelligent-storage.js` with three key fixes:

#### 1. Added International Name Protection Logic (Lines 1427-1450)
After LLM extraction, the code now:
- Calls `extractUnicodeNames` on the original user message to identify all international names
- Uses diacritic-aware normalization to compare extracted names with compressed facts
- Re-injects any missing names (with original special characters preserved) back into the facts

// FIX #759-CMP2: Protect international names with special characters
const inputNames = this.extractUnicodeNames(userMsg);
if (inputNames.length > 0) {
  const normalizeForComparison = (str) => {
    return str.normalize('NFD').replace(/[\u0300-\u036f]/g, '').toLowerCase();
  };
  
  const normalizedFacts = normalizeForComparison(facts);
  const missingNames = inputNames.filter(name => {
    const normalized = normalizeForComparison(name);
    return !normalizedFacts.includes(normalized);
  });
  
  if (missingNames.length > 0) {
    console.warn(`[EXTRACTION-FIX #759-CMP2] Re-injecting ${missingNames.length} lost international names:`, missingNames);
    facts += '\nContacts: ' + missingNames.join(', ');
  }
}
