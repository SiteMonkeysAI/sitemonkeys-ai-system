## Changes Made

### Area 1: PDF Upload Pipeline (`api/upload-for-analysis.js`)
Added PDF text extraction using `pdf-parse` (already in `package.json`). The fix inserts a PDF handler block before the plain text handler in `processFile()`. When a PDF is uploaded:
- Imports `pdf-parse` dynamically and extracts text
- Stores the result in `docxAnalysis` (same structure as DOCX/text) with `fullText` for document context injection
- Logs `[UPLOAD] PDF extracted: XXXX chars, X pages` on success
- Handles scanned PDFs gracefully with a descriptive message

### Area 5: GPT-4 Identity Leak (`api/core/orchestrator.js`, `api/lib/ai-processors.js`)
Added explicit identity guard in ALL system prompt locations:
- **Orchestrator** (`#buildSystemPrompt`): Added `IDENTITY (ABSOLUTE RULE)` section preventing "as an AI model developed by OpenAI" language
- **Eli processor** (`ai-processors.js`): Added identity rule to Eli's business validation system prompt
- **Roxy processor** (`ai-processors.js`): Added identity rule to Roxy's truth-first system prompt

### Area 3: Reasoning Scaffold Template Leaking (`api/core/intelligence/reasoningEscalationEnforcer.js`)
Stopped appending unfilled placeholder text (`_[Based on established patterns...]_`) to user-visible responses. The `enforceReasoningEscalation` function now flags missing steps for telemetry without modifying the response. The `buildReasoningScaffold` function remains intact but its output is no longer injected into `corrected_response`.

### Area 4: Business Survival Analysis "undefined" Values (`api/core/personalities/eli_framework.js`)
Fixed `#enhanceWithSurvivalMetrics` to only include the runway impact section when `runwayConsumed` is defined (i.e., actual calculated data is available). Previously, when burn rate data was missing, `runwayImpact` was still a truthy object but had undefined values, causing "Runway Impact: undefined (undefined of typical 6-month buffer)".

Also fixed Confidence Assessment (STEP 7) to skip when: external lookup succeeded (data already verified), or query doesn't require decision support (price lookups, news, document summaries don't need confidence framing).

### Area 9: "Start manual before automating" Leaking (`api/core/personalities/roxy_framework.js`)
Added domain and intent guards to the `manual_first` simplification. Now only added when: `analysis.requiresCalculation` AND `(domain === 'technical' || domain === 'business')` AND `(intent === 'problem_solving' || intent === 'decision_making')`. This prevents it from appearing on stock price queries.

### Area 8: Pricing Metadata in Responses (`api/lib/validators/anchor-preservation.js`)
Gated `#injectMissingAnchors` behind `process.env.DEBUG_ANCHORS === 'true'`. In production (default), the function returns the response unchanged. This prevents unrelated pricing data from memory (SiteMonkeys service tiers) from appearing as "(Key details: Pricing: $200, $5,100...)" in stock/general query responses.

### Area 7: Stale Price Memories (`server.js`)
Skip storing external real-time data (prices, news) as persistent memories. When `result.sources?.hasExternal` is true, all three storage paths (intelligent storage, supersession-aware, legacy fallback) now skip storage entirely. Prevents accumulation of conflicting stale price memories like three different ETH prices.

### Area 6: Document Query External Lookup (`api/core/intelligence/truthTypeDetector.js`)
Added short document reference patterns to `isDocumentReviewRequest()`. Queries like "Can you summarize what's in that document I just loaded" now return `DOCUMENT_REVIEW` type (even though they're short), which causes `isLookupRequired()` to skip external lookup. The `[DOCUMENT-LOAD→CONTEXT]` pipeline handles document queries instead.

### Area 2: Stock Price Fallback (`api/core/intelligence/externalLookupEngine.js`)
Changed the stock price handler from returning `[]` (empty/graceful degradation) to returning a Google News RSS fallback source. Stock queries now receive recent price-context headlines instead of a blank technical issue response or identity-leaking "as an AI model developed by OpenAI" message.
