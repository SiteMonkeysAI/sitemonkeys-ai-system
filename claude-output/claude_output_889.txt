## Changes Made

**File modified:** `api/memory/intelligent-storage.js`

**Location:** Inside `extractKeyFacts()`, after the CMP2 international name re-injection (line ~1731), before the final `return finalFacts`.

**What was added:** A minimum token floor check (Issue #889 fix) that runs after all compression and post-processing steps are complete:

```javascript
const MIN_TOKEN_FLOOR = 8;
const finalTokenCount = this.countTokens(finalFacts);
if (finalTokenCount < MIN_TOKEN_FLOOR) {
  const userMsgTokens = this.countTokens(userMsg);
  if (userMsgTokens >= MIN_TOKEN_FLOOR && userMsgTokens <= 50) {
    // Use original message instead of over-compressed facts
    finalFacts = userMsg.substring(0, 200).trim();
  } else {
    // Return '' to trigger storeWithIntelligence's existing fallback
    return '';
  }
}
```

**Why this works:**

1. The existing char-based quality gate (`approxTokens < 5`) catches obviously too-short fragments but uses an approximation that can differ from actual tiktoken counts. A string can pass the char-based gate (≥17 chars) yet still have fewer than 8 actual tiktoken tokens.

2. The new check uses `this.countTokens()` (which uses tiktoken when available, char-based as fallback) to enforce a strict 8-token minimum on the final compressed output — the exact threshold identified in the issue as necessary for meaningful embeddings and reliable retrieval.

3. **When below floor + user message is 8–50 tokens:** Uses the original user message verbatim. A self-explanatory message like "My salary is $95,000 at my current job" (~10 tokens) generates far better embeddings than the over-compressed "salary." (~2 tokens).

4. **When below floor + user message is outside 8–50 token range:** Returns `''`, triggering the existing `storeWithIntelligence` fallback that already uses `userMessage.substring(0, 200).trim()`.

**What was NOT changed (per issue constraints):**
- Retrieval thresholds — untouched
- Storage gates (`storeCompressedMemory` meaningless-content check) — untouched
- Supersession logic — untouched
- Embedding pipeline — untouched
- Keyword boost / priority tier logic — untouched
- Test files — untouched
- The existing `isTooShort` quality gate — untouched (additive fix only)

**CMP2/EDG3 safety:** International names (re-injected at line 1721-1729) and pricing lines (protected in `aggressivePostProcessing`) produce multi-token content that naturally exceeds the 8-token floor. The floor check is a safety net for cases where compression produces near-meaningless single-word fragments.
